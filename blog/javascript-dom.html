JavaScript & the DOM

Document Object Model - DOM

What Forms The World Wide Web

Most people have had exposure to web pages through the use of a device - smart phone, laptop or desktop computer or a tablet. When you open up a browser on your device, it will load, usually, a web page that connects your device to the greater internet (also known as the World Wide Web), and that page is referred to as a search engine. It is also just a web page that is interactive. That web page, like all other pages, is built on a consistent foundation known as a document.

The document is the consistent basis for every webpage you view. But the document itself is just a blank canvas with a bunch of rules that you can't see. This document is a sort of scaffolding that is described as an object. An object in coding is a way of describing in code what a real world object might look like. The document is such an object. Objects are are model or representation of that real life thing. So the foundation of a web page is referred to as the Document Object Model or DOM.

JavaScript's Control of the DOM

What you see on the web is likely instructions from three interrelated programming languages that reside on the DOM that determine the structure and content (HTML); how the page is organized and styled (color, fonts, placement, size, etc.)(CSS); and how you can interact with with the page (JavaScript).

A web page can be created with HTML and CSS alone. These web pages are not very efficient and are rather dull. JavaScript, on the other hand, can create interaction for the user, change the HTML and CSS and create depth to the web page. In a way, HTML and CSS are like a flat paper printer. A 2D printed page may look nice, convey a message and important information but not much else. JavaScript would be taking the concept of printing on 2D paper and creating a 3D print that you could interact with.

So while HTML and CSS are important, JavaScript is what will control the what interation the DOM provides.

Control Flow and Loops

Like things that are routine in our lives, coding for a website relies on a certain flow and, in some cases, repetition. For example you are in charge of preparing a meal for four people and have to choose what meal it is. First you need to decide what meal is appropriate given the season. If its winter you might make a stew. If its summer you might make a salad. If one person is vegan you may need to make one of the meals with no animal products. In programming this is control flow, which is done using logic structures such as an if-then statement (see below).

Once the decisions are made for the meal and assuming all four have the same preferences, you will need top prepare four meals that are the same. In coding, this is where loops come in. You instruct JavaScript to repeat the same action for all of the meals. This is typical done through a for loop which comes in a variety of syntaxes. Perhaps the most common is the forEach loop.

JavaScript DOM Interactions

One of the powers of JavaScript is its ability to interact with the Document Object Model (DOM). However, to access the DOM there must be a location to target its interaction. These targets are created through accessing HTML elements (tags) that are on the page and have either a class (classes group things together) or an id (ids are unique).

For example, you have a button on the page that you want to change the label when that button is pressed. To allow JavaScript find this specific button you would add an id to its opening element/tag and "query" the document to set it as the target for JavaScript to make the change. Getting Data From Objects & Arrays

Both objects and arrays organize and hold data that can be accessed and used. An object holds information through pairing a key with a value: let obj = {key: value}. An object can hold as many keys as there are values that form the object.

Arrays are more like an ordered list of whatever values you want to include. Each value has an index attached to it that refers to its place in the list. Indexes start counting from 0 instead of 1. So the first position in an array is 0 not 1.

Because they are structure differently, the way to access their data differs. For objects use . notation: let objValue = object.key to set the variable objValue to the value held by the object key.

For arrays you use the name of the array followed by square brackets containing the index number of the data you want: let arrValue = array[index]. So to set the variable arrValue to the data held in the third position of the array the code would look like this: let arrValue = array[2].

Functions

Functions are routines that you create for JavaScript to perform when requested. Functions are reusable code. This means that you can write it once and apply it anytime and anywhere you want.
